## Design Principles

### 1. Interruptions are normal
- âœ… Auto-save in all forms
- âœ… "Continue where you left off" functionality
- âœ… Never lose user data

**Why:** Life happens. Kids cry, phones ring, doors knock. Good design doesn't punish interruptions.

### 2. Mistakes are okay
- âœ… Easy undo
- âœ… No aggressive confirmation dialogs
- âœ… "Are you sure?" â†’ "You can undo this later"

**Why:** Errors are human. Good design forgives them.

### 3. Minimal cognitive load
- âœ… One thing at a time
- âœ… Clear progress indicators
- âœ… No unnecessary choices

**Why:** Tired, stressed, or neurodivergent users need clarity, not complexity.

### 4. Works in stressful situations
- âœ… Large click targets (easy to hit)
- âœ… High contrast
- âœ… No hidden actions

**Why:** Hands shake, vision blurs, time is short. Design shouldn't add stress.

### 5. Safe to use in public
- âœ… Doesn't show sensitive data by default
- âœ… Privacy-first design
- âœ… Non-intrusive notifications

**Why:** Many people use services in public placesâ€”buses, cafes, workplaces.

### 6. Technical accessibility
- âœ… WCAG AA (or AAA where possible)
- âœ… Keyboard navigation
- âœ… Screen reader optimized
- âœ… Semantic HTML

**Why:** Legal requirement, but more importantly: human necessity.
